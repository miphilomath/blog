Basics_of_packaging

February 13, 2019 3-minute read

Ever installed any software in you linux based operating system? Yes? So might have installed it using one of the two methods, either by using your system package manager, or by downloading and installing .deb/.rpm packages.

So, installing software in linux can be done in 2 ways:

  1. Using system package manager, which apt for debian based or dnf for redhat based operating system by apt install <software>.
  2. Using .deb or .rpm packages downloaded from the software respective website by apt install <.deb file>

So, how does these methods work, and what are the differences between them? Before finding answers to above questions, let’s find more about on basics.

Packages

Each and every software depends on various other softwares, programs or libraries (dependencies), that is required by that software to function properly and to install any software we need to install all those dependent softwares and libraries as well. For users, installing these many dependent softwares could be frustrating and there has to be some way to install software easily, so that they can dedicate their time more on their work rather than configuring and installing the required software.Software packages ease the process of installing software on your system so that user can do his work on the software not waste time on its installation.

Packages contain meta data, such as the software’s name, description of its purpose, version number, vendor, checksum, and a list of dependencies necessary for the software to run properly.

So, in short, software packages or packages is like packing the software itself and all its dependent softwares in single file known as package with extensions .deb or .rpm depending on the package manager it uses.

Package Manager

Never heard of package manager?

Package Managers are softwares that manages software packages, automates the process of installing, upgrading, configuring and removing computer programs in a consistent manner (i.e. in a way that don’t leave your system in an inoperable state and handles installation of all dependent software by itself).

Upon installation, meta data is stored in a local package database. Package managers maintain a database of software dependencies and version information to prevent software mismatches and missing prerequisites. On Debian based operating system, “aptitude” package manager is used while Red hat based operating system uses RPM.

Difference between both the installation methods!

Ultimately, in both the methods, software is installed using the software packaged for the operating system i.e. .deb or .rpm file

So, why use two different methods? Where is the .deb file while installing from official repository? What is the advantage of one over other? Calm down! We will find answers to all these questions.

First, both methods are used for different purpose

Second, the software installed using official repository downloads the software package for that particular software from that operating system official repository, and then installed later on.

those .deb files are stored in /var/cache/apt/archives directory of your operating system.
