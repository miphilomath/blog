Git Init

January 25, 2020 3-minute read

This installation was done during my attempt to contribute to git project. Alas! No luck

This post documents the procedure to install git from its source code, problems faced aka challenges and their solutions that worked for my system. Installing git from source is not as easy as sudo apt install git and doing so, you will realize the importance of packages available in official repositories to save our time and effort just to install the software.

System details:

Os: Linux Mint Sylvia

Model: Dell Inspiron 3543

Kernel: 4.10.0-42-generic

Ram: 4GB

CPU: Intel I3 4th gen

Following are the steps to install git on your system from its source.

So, by now, you might have guessed, the first step would be to download its source code to install.

\1. Download source code of GIT from github mirror or their git repo here using web interface or git clone utility as: git clone https://github.com/git/git`.`

If you have downloaded the tar file from the web interface, untar it and enter into its directory using cd utility.

Note: Now, to understand a little, what you are going to do next, read INSTALL instruction file in there. If you want to keep your currently installed git intact, install the git from source to another directory (other than from user’s /bin directory).

I installed my git on /usr/local/bin directory but you can install it anywhere you want. But if you install in directory not pointed by your system PATH variable, make sure to enter whole path of the installed git directory (i.e. from source) when executing it. In short, the directory chosen by you,k is the place where the git will be installed locally.

I used profile feedback option to install git on my system which might be faster at times in cpu intensive tasks while trading my compiling and installation time. You might chose to install it without this option. (Follow INSTALL file in the directory)

Make sure you have enough time (about 3-4 hours) to install git (with profile feedback) and tackle various challenges that you might encounter (can be different for your system, but stackoverflow can help)

\2. make prefix=/path_to_directory profile

In my case, path_to_directory was /usr/local, to install git on /usr/local/bin

profile is used for profile feedback option.

\3. make prefix=/usr/loca/ PROFILE=BUILD install

This will install git on the prefixed directory.

Challenges:

There were many libraries missing on which git is dependent (details below) and may throw error while installing. I am documenting few of them. This blog might have solution to your error and might be helpful.

1: libssl-dev missing: sudo apt install libssl-dev

2: libcurl missing: `sudo apt install

Third challenge: asciidoc missing. (http://www.methods.co.nz/asciidoc/INSTALL.html#X1)

fourth: xsltproc missing. (https://askubuntu.com/questions/462333/where-can-i-download-xsltproc-that-is-compatible-with-ubuntu-12-04#46234, http://xmlsoft.org/XSLT/xsltproc.html

fifth: xmlto missing: sudo apt install xmlto

sixth: docbook2x-texi: not found (http://docbook2x.sourceforge.net/, https://sourceforge.net/projects/docbook2x/files/docbook2x/0.8.8/docbook2X-0.8.8.tar.gz/download?use_mirror=excellmedia, https://packages.debian.org/jessie/amd64/docbook2x/download)

install it using sudo make install….

make profile=/…. PROFILE=BUILD install

seventh: perf failed, ,can’t locate json.pm in /usr/local/bin but in /usr/bin so if are installing as global, then you might not encounter this error. INstall perl module JSON: sudo perl -MCPAN -e ‘install JSON’

Note: Using multiple jobs would be helpful for you. depending on the number of cores, your system support, add attribute -j 2/3/4 or -j 20 if you own a i9 :P

Wait for 3 hours :P

DONT QUIT, if you feel frustated, (cause I am :/)

So many missing libraries, It took an hour just to complete all the dependecies :D

Never realized the importance of packaging before, they really help us to work easily :)

Important: First check their official docs here https://git-scm.com/book/en/v2/Getting-Started-Installing-Git
